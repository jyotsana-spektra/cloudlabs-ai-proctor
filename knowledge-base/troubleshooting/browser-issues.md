# Browser Issues

## Overview

This article provides troubleshooting steps for browser-related issues that may prevent CloudLabs labs, Azure Portal, or Microsoft Fabric from functioning correctly.

---

# Symptoms

Users may experience one or more of the following:

- Blank page after opening the lab
- Buttons do not respond
- Infinite loading spinner
- Browser freezes or crashes
- Portal elements fail to load
- JavaScript errors
- Unsupported browser message

---

# Possible Causes

Common causes include:

- Outdated browser version
- Corrupted browser cache or cookies
- Unsupported browser
- Browser extensions interfering with page rendering
- Pop-up blocker preventing required windows
- Network connectivity issues

---

# Resolution Steps

## Step 1 – Use a Supported Browser

Recommended browsers:

- Microsoft Edge (Latest)
- Google Chrome (Latest)

---

## Step 2 – Refresh the Page

Perform a hard refresh.

Windows:

```
Ctrl + Shift + R
```

Mac:

```
Cmd + Shift + R
```

---

## Step 3 – Clear Cache and Cookies

Clear:

- Cache
- Cookies
- Site Data

Restart the browser afterwards.

---

## Step 4 – Disable Extensions

Temporarily disable:

- Ad blockers
- Privacy extensions
- Script blockers

Reload the application.

---

## Step 5 – Try an Incognito/InPrivate Window

Open the application in a private browsing session.

---

## Step 6 – Check Internet Connection

Verify:

- Stable internet connection
- VPN is disabled if not required
- Corporate proxy is not blocking Microsoft services

---

# Verification

The issue is resolved when:

- The portal loads successfully.
- All controls are responsive.
- Labs can be completed normally.

---

# Escalation

Escalate if:

- Browser crashes repeatedly.
- Multiple browsers exhibit the same behavior.
- Issue persists after cache clearing.

Include:

- Browser name
- Browser version
- Operating system
- Screenshot
